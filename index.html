Why API Management?

So why API Management... why are we here today? Let's start with...

A Debate...

Services vs APIs

Service-Oriented Architecture

Developers focused on building "services"

What is a "service"?

A means by which providers codify business domain capabilities into reusable units.
Roughly 20 years ago, Service-Oriented Architecture began gaining traction. Developers began building these things we called "services". So what is a service? Basically we're building reusable components that represent our business domain what are available for our software applications to consume. Developers were able to call to each other's services remotely.

Pitfalls

Services everywhere

Lack of documentation

Usage control (security, throttling, limits, who's calling?)

Issue resolution

Services all over the place. Not cataloged. No way to get a grip around what is available. No good definitions around what these services do, what they different values of the schema refer to, etc. The means to access each service is inconsistent. No consistently way to provide feedback to service developers on issues that occur.
So let's shift our SOA mindset to...

API-Oriented Architecture

Complimentary to SOA

Allow services to be more easily consumed and managed

  • The focus, or intent of an API-Oriented Architecture is to ((((Read last fragment)))

An "API" is:

A means by which services are packaged, productized and shared in an easy-to-use format.

API's potential is maximized when it is managed.

Really what we're doing here is adding a wrapper around our services to give them better definition in an effort to make them easier to use and maintain.

API Management Core Components

  • Gateway
    • Proxy API requests/responses
    • Policies (throttling, security, etc)
    • Transform requests/responses
    • Handle authentication, authorization
    • Caching and analytics collection
  • Publishing tools
    • Manage API lifecycle (Creation, versioning, revisions, retirement)
    • Managing access and policies
    • Developer testing & debugging

API Management Core Components, cont'd

  • Developer portal/API store
    • Self-service access to APIs
    • Interactive console for test-driving
    • Ability to subscribe, gain access keys
  • Reporting and analytics
    • Monitor and report on API usage and loads
    • Raise alerts
  • Monetization
    • Charging for API access
